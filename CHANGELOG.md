# CHANGELOG

История изменений проекта в последовательном порядке по итерациям.
Подробные инженерные инварианты и архитектурные решения фиксируются в `CODEX.md`.

## 2026-05-05

- Собран базовый MVP MV3-расширения для DOM capture, ограниченного network capture и передачи `latestCapture` в extension viewer.
- `fetch`-instrumentation переведён в режим композиционной обёртки поверх текущего patched `window.fetch`.
- После security review захват сети ограничен armed-доменами: пользователь сначала включает сбор для текущего домена, затем перезагружает страницу и только после этого получает полный ранний буфер запросов.
- Добавлен TTL для `latestCapture`, чтобы временные данные не лежали в `storage.local` бесконечно.
- Усилен внутренний канал page→content и расширена редактировка чувствительных заголовков и очевидных секретов в body.
- Добавлен popup-меню управления: toggle armed/unarmed для текущего домена, запуск DOM capture, копирование `latestCapture`, очистка и открытие viewer.
- Popup и viewer приведены к более выразительному визуальному стилю; технические тексты переписаны в более короткую и понятную продуктовую подачу.
- Popup дополнительно ужат до компактного окна без лишних текстов и вертикального скролла; формулировки в нём упрощены и сокращены.
- В режиме выбора элемента клик теперь перехватывается расширением и не должен запускать редиректы или другие page-level click handlers.
- Открытие viewer из модального окна теперь автоматически сохраняет текущий capture; пользователю больше не нужно отдельно нажимать сохранение перед просмотром.
- Временные in-page подсказки теперь автоматически скрываются по таймауту и не остаются висеть после завершения действия.

## 2026-05-06

- Добавлен эвристический `elementRecipe`: viewer теперь показывает API-последовательность выбранных запросов, request body, shape JSON-ответа и поля ответа, совпавшие с видимым текстом выбранного элемента.
- DOM capture расширен селектором, безопасным набором атрибутов, цепочкой предков, текстовыми фрагментами и `previewHTML` для отдельной визуализации выбранного элемента.
- `fetch(init.body)` и `XHR.send(body)` теперь сохраняют best-effort `requestBody` с тем же принципом локального редактирования очевидных секретов; тело `Request`-объекта без `init.body` сознательно не читается.
- Viewer получил sandboxed HTML-превью выбранного элемента с CSP `default-src 'none'`; превью не должно загружать внешние ресурсы и остаётся локальным отображением захваченного фрагмента.
- Дизайн viewer приведён к более утилитарному интерфейсу для чтения API-рецепта, DOM-снимка и сырого JSON без декоративного фона.
- `elementRecipe` повышен до `cloneSpec` v2: выбранный виджет раскладывается на `domFacts`, ответы выбранных API — на `responseFacts`, а между ними строятся `bindings` с confidence, причинами совпадения и JSON-path.
- Ранжирование кандидатов API теперь учитывает не только время и полный текст, но и совпадения нормализованных DOM-фактов с JSON-листьями ответов; это лучше подходит для таблиц, карточек и списков.
- Добавлен bounded DOM mutation trace для armed-доменов: локально сохраняются недавние текстовые изменения DOM, чтобы binding мог получить evidence “API-ответ → DOM-значение появилось/изменилось”.
- Для `fetch`/`XHR` сохраняется локальный best-effort call stack места вызова, чтобы viewer мог показывать frontend-источник запроса без подключения DevTools/CDP.
- Viewer получил кликабельную “Карту значений”: клик по найденному значению показывает endpoint, JSON-path, confidence, причины совпадения, DOM context, sibling fields и render evidence.
- Долгие операции анализа теперь показывают staged progress: снятие DOM, извлечение DOM-фактов, сравнение API-ответов, сборка `cloneSpec`, сохранение и планирование TTL-очистки.
- Исправлен crash при извлечении дат из DOM-фактов: `matchAll` теперь получает отдельный global RegExp, а неглобальный `DATE_PATTERN` остаётся только для `.test()`.
- Уточнён скоринг provenance: “5 дней” классифицируется как duration, строки вроде `evaluation_5plus_web` больше не превращаются в число `5`, а слабые одиночные numeric-match без семантического контекста отбрасываются.
- Viewer показывает альтернативные API для одного DOM-значения, чтобы пользователь видел несколько возможных источников и не принимал первый высокий score как единственно верный.
- Лимит выбранных API-кандидатов увеличен с 5 до 8, чтобы в неоднозначных виджетах показать больше похожих backend-ответов без повторных запросов к API.
- Добавлены `apiDependencies` для sequence-диаграммы вызовов: связь строится только если значение из ответа раннего захваченного API найдено в URL/body/header более позднего захваченного запроса; без таких связей viewer не показывает диаграмму.
- Сырые JSON/debug-блоки во viewer теперь свернуты по умолчанию через “Подробнее”, чтобы основной рецепт оставался компактным.
- Добавлена локальная JSON-выгрузка render-spec: пользователь выбирает категории данных и отдельные поля объекта, а viewer формирует `dataModel.schema`, `fields`, `apiCalls` и `apiDependencies` для внешнего рендера без отправки данных наружу.
- HTML-превью виджета усилено расширенным набором безопасных computed inline styles, включая layout, flex/grid, размеры, тени и типографику.
- Секция сырого capture JSON принудительно закрывается при каждом новом рендере viewer, а в шапку добавлена короткая цитата Вороваек.
- JSON-экспорт переориентирован на основной `apiRecipe`: порядок вызовов API, зависимости между ними, `requiredData` с типом, описанием, примером значения и точным JSON-path в ответе нужного API теперь находятся в верхней части payload.
- Минимальная JSON-выгрузка теперь включена по умолчанию: `apiRecipe.minimalDataMap` группирует нужные поля по API URL и JSON-path, а структура ответа, request body/headers, DOM и debug стали явными опциональными чекбоксами.
- Flow копирования capture в popup переработан: `latestCapture` остаётся одноразовым сообщением для viewer, а повторное копирование и повторное открытие viewer используют локальный `copyableCapture` с тем же TTL и общей очисткой.
- Viewer переведён на вкладочную структуру: сверху показываются превью, метрики и таблица “Минимальный рецепт”, а подробности разнесены по вкладкам `Поля`, `API`, `Export`, `DOM` и `Debug`.
- JSON-выгрузка сужена до строгого `element-export`: в payload остаются только верстка выбранного элемента и связанные API с JSON-path нужных данных; `responseShape`, сырые ответы, DOM-контекст и debug исключены из export.
- В таблице “Минимальный рецепт” значение из выбранного DOM-элемента теперь показано отдельно от raw-значения поля в API-ответе; минимальный export сохраняет `displayValue` и `responseValue` для связи UI-значения с `JSON path`.
- Для числовых primary values вроде “5 дней” усилено связывание DOM↔API: DOM-факты получают общий текст выбранного виджета, duration может матчиться с числовым API-полем вроде `5.000`, а viewer восстанавливает такие пропущенные строки из уже сохранённых `responseFacts`.
- JSON-выгрузка больше не содержит CSS-слой выбранного элемента: из экспортируемого HTML удаляются inline styles, class/part-атрибуты, `<style>` и stylesheet `<link>`, а CSS-селектор элемента исключён из payload.
- JSON-выгрузка теперь автоматически добирает companion values из того же объекта ответа: если выбран текстовый label вроде `shortText = отпуска`, рядом найденное числовое значение дней попадает отдельным data field с собственным `JSON path`, `displayValue` и `responseValue`; для старых captures primary values дополнительно извлекаются из `element.textPreview`.
- Анализ захваченных API-ответов расширен для больших JSON: сохраняется до 512KB ответа, локальный network buffer вырос до 60 записей, candidate list до 12 запросов, JSON обход теперь смотрит до 6000 узлов, 250 элементов массива/ключей объекта и хранит response facts по отдельным лимитам на ответ и на весь рецепт.
- Viewer добавил fallback raw-response scan: если `responseFacts` не содержат видимые числа вроде `22 дня` и `5 дней`, они ищутся напрямую в сохранённом `capture.network[].responseBody` выбранного API, scoped к тому же объекту ответа; числовые значения с нулевой дробной частью `22.0/22.00/22.000/22.0000` сопоставляются с `22 дня`.
- Исправлен `QuotaBytes quota exceeded` при открытии viewer: большой raw response используется для анализа до сохранения, а в `chrome.storage.local` пишется compact capture с урезанными debug-preview; `latestCapture` теперь хранит лёгкую ссылку на `copyableCapture`, чтобы один capture не занимал quota дважды, а при ошибке quota включается tight-компактация.
- В `cloneSpec` добавлены `evidenceTimeline` и `analysisDiagnostics`: viewer показывает последовательность выбора элемента, захваченных API, связей DOM↔JSON, dependency edges и DOM render evidence, а также объясняет, почему видимые значения не удалось уверенно связать с выбранными API.
- API-последовательность в рецепте теперь дедуплицирует одинаковые endpoint-вызовы по нормализованному URL с volatile query-параметрами вроде `rquid`, `traceId`, timestamp и cachebuster; viewer показывает количество свёрнутых дублей без повторных запросов к API.

## 2026-05-07

- Popup упрощён и теперь показывает компактный статус последнего захвата: выбранный DOM-узел, количество выбранных API и время capture.
- Кнопка `Скопировать захват` переориентирована на единый `captureBundle`, а не на старый полный capture JSON.
- В модалке выбора API количество кандидатов увеличено до `20`, чтобы нужные запросы чаще попадали в список.
- Модалка выбора API теперь показывает краткий preview ответа рядом со score и метаданными запроса, но без раздувания интерфейса полным body.
- Viewer упрощён до трёх основных задач: проверить DOM preview, увидеть выбранные API и настроить экспорт с preview итогового payload.
- В DOM capture добавлен `cleanHtml`: структура выбранного поддерева без `class`/`style`/`part` и без инлайна base64, с сохранением URL-атрибутов.
- В сохранённый capture добавлены `captureBundle` и `captureSummary`, чтобы popup и viewer читали один и тот же канонический payload.
- В `CODEX.MD` добавлено обязательное правило: после любых изменений Codex обязан обновлять документацию и `CHANGELOG.md`.
- Viewer export получил отдельный режим `API types`: он строит компактную структурную типизацию выбранных API-ответов прямо из сохранённого `responseBody`, чтобы LLM можно было кормить схемой ответа без полного raw payload.
- Дефолтный экспорт `Всё вместе` теперь отдаёт `DOM clean + API types`; полный `API` и сырой `DOM raw` оставлены отдельными явными режимами.
- Полный capture теперь сохраняется отдельно от `chrome.storage.local` во внутреннем фоне расширения; popup и viewer сначала пытаются читать именно его, а compact-версия в `storage.local` остаётся fallback на случай quota или недоступности полного payload.
- Viewer больше не молчит о деградации: если открыт compact fallback или если в request/response/DOM уже есть `...[truncated]`, сверху показывается явное предупреждение.
- `cleanHtml` теперь вычищает inline `data:image;base64` из экспортируемой структуры, а sandboxed preview, наоборот, сохраняет безопасные `data:image` URL, чтобы локальные иконки могли отрисоваться без внешних запросов.
